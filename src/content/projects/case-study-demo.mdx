## What This Is

This is a **demo entry** for the new `CaseStudyLayout`. It lives alongside the existing `ProjectLayout` — no migration needed.

## How to Use It

For any new project (or when you feel like rewriting an old one), add these optional frontmatter fields:

- `context` — Why did you build this? What problem were you solving?
- `role` — What did you actually do? Frontend only? Design + code?
- `decisions` — Array of strings. Why this layout, this color, this interaction?
- `outcome` — What happened? Metrics, user feedback, or just "it worked."

If you omit all of them, the layout falls back to a cleaner version of the old project page.

## Technical Notes

The layout renders the case study fields **above** the MDX content. The content area is still a two-column grid with `prose` styling. The stack sidebar stays consistent with the old layout.

If a project doesn't have a `cover.webp`, set `hasImage: false` (or omit the screenshot section from the layout — it already checks for this).
